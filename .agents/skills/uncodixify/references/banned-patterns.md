# Banned Patterns

Treat these as red flags. Remove them before adding more style.

## Structure

- dashboard hero strips that behave like landing pages
- KPI-card grids as the first instinct for operational screens
- fake analytics, fake charts, or placeholder trend widgets
- oversized empty sections created only to look expensive
- right rails, glass shells, or floating sidebars with no product reason

## Visual Language

- glassmorphism as the default surface treatment
- thick gradients used to fake taste
- border radii that are large on everything at once
- pill buttons, pill tabs, pill badges, and pill filters everywhere
- decorative glows, blur haze, conic gradients, or colored shadows

## Copy

- eyebrow labels such as `LIVE PULSE`, `TEAM FOCUS`, or similar decorative framing
- dashboard copy that explains how premium or clear the product is
- filler subtitles whose only purpose is atmosphere
- badges or labels that exist only to sound alive or intelligent

## Motion

- hover transforms as the default affordance
- bouncing, sliding, or drifting animation on ordinary controls
- shimmer effects, animated progress fills, or ornamental motion in internal tools

## Components

- cards that all share the same rounded rectangle treatment
- icons inside decorative colored chips without functional meaning
- tag badges on every table row when plain text would read better
- section headers that use a `small` label plus a big marketing sentence

## Review Questions

- Would this still make sense if all gradients and blur were removed?
- Is this component carrying real information, or only trying to look premium?
- Would a designer keep this, or delete it in the first cleanup pass?
