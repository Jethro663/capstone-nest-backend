Backend follows NestJS module/service/controller structure with DTO validation, Drizzle schema files, and envelope-style API responses. Frontend uses app router, service layer wrappers under `src/services`, typed interfaces under `src/types`, and shadcn-style UI components. Existing AI architecture favors minimal-disruption extension: NestJS keeps auth/RBAC and proxies to ai-service; LXP/performance are event-driven; teacher control is preserved through review/apply and intervention assignment flows.
